# 1. Référentiel métier (Source of Truth) ✅

À figer maintenant.

Contient :

### A. Classification compétitions

Règle simplifiée retenue :

```text
3 digits  => National (N)
4 digits  => Régional (R)

Numéros réinitialisés / incrémentés chaque année.
Le type découle du numéro (pas besoin d'un flag manuel).
```

👉 devient règle métier officielle.

---

### B. CompetitionRegistry.php

Doit contenir :

```text
- niveaux (N / R)
- disciplines
- catégories / sections
- règles KPI
- règles accès
- statuts concours
- mapping règlements FPF
```

Le déclarer explicitement :

```text
CompetitionRegistry = référentiel métier central
```

Pas simple config, mais dictionnaire métier.

# 3. Architecture figée V1 (à sanctuariser)

On acte ces principes :

```text
Import -> normalise données
Registry -> règles métier
SQL KPI -> calculs officiels
Controllers -> orchestration uniquement
Views -> aucune logique métier
```

Règle d’or :

```text
Pas de logique métier dans les contrôleurs.
Pas de règles KPI dans les vues.
Pas de duplication hors Registry.
```

C’est fondamental.
